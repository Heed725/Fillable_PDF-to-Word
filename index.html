How it works

1. Detect fields

We inspect interactive PDF widgets and capture field names, types, and current values.

2. Build a Word sheet

The app creates a Word document with a clean table so values are easy to review and update.

3. Preserve metadata

Hidden metadata keeps the field mapping intact so the edited document can flow back into the PDF.

4. Refill the template

When you upload the edited document and source PDF, matching form fields are written back into the PDF.

What to expect

Best inputInteractive PDFs created with real form fields.
OutputA .docx file for editing, or a filled .pdf when converting back.
PrivacyNo upload step. Files stay in the current browser session.

Editing tips

  • Keep the generated field names in place so the app can map values back correctly.
  • For checkboxes, use values like "Checked", "Yes", "True", or "1" to mark them on.
  • If a field does not refill, make sure you used the original PDF template and not a flattened copy.
All processing happens in your browser. No uploads. No server required.
